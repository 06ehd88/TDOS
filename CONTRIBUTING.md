# Contributing

## Branch policy

- `main`: approved or explicitly identified baseline content
- `develop`: reviewed engineering work not yet baselined
- `feature/*`: isolated feature or specification changes
- `fix/*`: corrective changes

## Change discipline

Every change shall identify:

- affected controlled asset,
- baseline impact,
- runtime impact,
- verification impact,
- evidence,
- decision authority.

Do not replace `NOT AVAILABLE` or `PLACEHOLDER` content with assumptions.
