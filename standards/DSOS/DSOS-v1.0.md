# Development Simulator Operating Standard (DSOS) v1.0

## Status

Approved Operating Standard

## Purpose

Defines how the TDOS Development Simulator operates.

## Canonical input rule

The current approved TECR is the sole canonical engineering candidate input.

Raw reference-mission reports, candidate packs, lessons learned and engineering notes shall be normalized into TECR before evaluation.

## Session lifecycle

```text
Engineering Baseline Loading
  ↓
Registry Synchronization
  ↓
TECR Candidate Selection
  ↓
Controlled Asset Discovery
  ↓
FIP Evaluation
  ↓
RQC Assessment
  ↓
Governance Compatibility Check
  ↓
REII Routing
  ↓
Engineering Recommendation
  ↓
Engineering Session Report
  ↓
Registry Update Actions
  ↓
Session Closure
```

## Mandatory outputs

- Engineering Session Report
- Registry Update Actions
- Open Engineering Questions
- Integration Recommendations

## Prohibitions

The simulator shall not approve architecture, authorize implementation, modify the baseline, or evaluate raw unnormalized candidate documents.
