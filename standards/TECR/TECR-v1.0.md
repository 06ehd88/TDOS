# TR Engineering Candidate Registry (TECR) v1.0

## Status

Canonical Engineering Candidate Registry

## Canonical input rule

TECR is the sole canonical engineering candidate input for the Development Simulator.

## Registry schema

| Field | Description |
|---|---|
| Candidate ID | Unique identifier |
| Title | Canonical candidate title |
| Category | Capability, standard, infrastructure, verification, etc. |
| Origin | Reference mission or engineering source |
| Evidence | Supporting evidence references |
| Finding | Engineering problem statement |
| Expected Benefit | Engineering value |
| Duplicate Sources | Merged source documents |
| FIP Classification | Type A–E |
| Priority | P0, P1, P2 |
| Affected Controlled Assets | Impacted assets |
| Preferred Integration Target | Target artifact |
| Status | Registry state |
| Decision | Authorized disposition |
| Notes | Additional context |

## Registry state

The Kastamonu candidate packages have not yet been fully normalized into canonical entries. See `registry.csv`.
